Description du projet

L’agence souhaite publier la nouvelle version de son site vitrine. Il s’agit d’un site one-page.

L'HISTOIRE

Le design a été validé, et un premier développeur freelance a commencé l’intégration il y a quelques semaines.
Malheureusement, ce freelance a dû abandonner le projet pour raisons personnelles.
Le site est fonctionnel, mais quelques bugs entravent le bon usage par les visiteurs. 724events a donc fait appel à moi pour finaliser le travail.
J'ai également rencontré Jean-Baptiste, le directeur Marketing de l’agence, en visioconférence pour qu’il me présente plus précisément le projet.

MON APPROCHE

Dans ce projet, j'ai dû débugger et finaliser le développement d'un site one-page.
J'ai utilisé les Chrome DevTools pour identifier et comprendre les problèmes et bugs signalés dans le site web actuel.
Après avoir identifié les problèmes, j'étais chargée de résoudre ces bugs et problèmes. Cela impliquera l'utilisation de mes compétences en JavaScript, ainsi que des outils spécifiques à React, comme React Developer Tools.
J'ai également dû compléter le cahier de recette. Ce document détaillera tous les scénarios de test nécessaires pour vérifier le bon fonctionnement du site, incluant des tests unitaires et fonctionnels.
Pour la gestion du projet, j'ai utilisé Yarn comme gestionnaire de packages et Node.js pour exécuter du code JavaScript côté serveur.
Ces outils m'ont aidé à gérer les dépendances et à exécuter des tests localement. Enfin, j'ai versionné mon travail avec GitHub et utilisé Visual Studio Code comme éditeur de code pour développer et débugger le site.

L’entreprise souhaite développer un site Internet qui permette aux usagers de trouver des hébergements et des activités dans la ville de leur choix.

L'HISTOIRE

J'étais chargée d'intégrer l'interface du site avec du code HTML et CSS. Pour cela, j'ai travaillé en étroite collaboration avec Sarah, la CTO, et Loïc, l’UI designer.

MON APPROCHE

Dans ce projet, j'ai créé la page d'accueil d'une agence de voyage en utilisant HTML et CSS. Ma mission principale était d'intégrer l'interface responsive de leur site. J'avais à disposition les maquettes Figma pour mobile, tablette et desktop, ainsi que les images et une note de synthèse sur les spécificités du projet. J'étais amenée à créer mes propres composants d'interface avec HTML et CSS afin d'avoir un rendu correspondant à la maquette. Cela m'a donné une compréhension plus profonde de la manière dont le code est structuré et stylisé pour répondre aux exigences de design.

Description du projet

Création d'une page web dynamique avec JavaScript pour le site d'une architecte d'intérieur.

L'HISTOIRE

Travaillant comme développeuse front-end pour l'agence ArchiWebos et ayant terminé mon dernier projet avec un peu d'avance, j'ai été envoyé en renfort comme développeuse front-end d'une équipe qui travaille sur la conception du site portfolio d'une architecte d'intérieur.

MON APPROCHE

Ce projet m'a engagé à créer une page web dynamique pour Sophie Bluel en utilisant JavaScript et en communiquant avec une API. J'ai travaillé sur la partie Front-End du site, où j'ai développé la page de présentation des travaux de l'architecte à partir du HTML fourni. Cela impliquait la gestion des événements utilisateurs et la manipulation des éléments du DOM avec JavaScript. J'ai également créé la page de connexion pour l'administrateur du site ainsi qu'une modale permettant d'uploader de nouveaux médias.
